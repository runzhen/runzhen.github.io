什么是堆#

堆是一棵具有特定性质的二叉树。

  1. 堆首先必须是一棵完全二叉树；
  2. 这棵完全二叉树中，任意一个结点的值都大于等于它的左右孩子结点的值，这样的堆称为大顶堆（也叫最大堆，Max-Heap）；或者任意一个结点的值都小于等于它的左右孩子结点的值，这样的堆称为小顶堆（也叫最小堆，Min-Heap）

因为它是完全二叉树，所以通常在写代码的时候，我们会用一个数组表示树的元素，而且会把第 0 号空出来，这样：

  1. 假设 parent index 为 i，左节点 idx = i * 2，右节点 idx = i * 2 + 1 。
  2. 假设某个节点 index = i，其父节点 idx = i / 2 。

什么是优先队列#

Priority Queue 是一种数据结构，普通的 Queue 是先进先出，而 PQ 能根据每个元素包含的优先级，选择下一个出队的元素。堆 （Heap）通常是实现 PQ 的底层数据结构，但并不意味着 PQ 一定是堆。

通常有以下应用

  • 数据压缩：赫夫曼编码算法；
  • 最短路径算法：Dijkstra算法；
  • 最小生成树算法：Prim算法；
  • 事件驱动仿真：顾客排队算法；
  • 选择问题：查找第k个最小元素；

Golang Heap#

Go 的 container/heap包提供了一个接口，

type Interface interface {
	sort.Interface
	Push(x interface{}) // add x as element Len()
	Pop() interface{}   // remove and return element Len() - 1.
}

要想实现一个 heap，用户需实现这个接口的 Push，Pop 函数，外加 sort 接口的所有函数。

如下所示，

package main

import (
	"container/heap"
	"fmt"
)

// 定义一个数组 
type PQ []int

// 实现 sort 接口
func (h PQ) Len() int           { return len(h) }
func (h PQ) Less(i, j int) bool { return h[i] < h[j] }
func (h PQ) Swap(i, j int)      { h[i], h[j] = h[j], h[i] }

// 实现 Push
func (h *PQ) Push(x interface{}) {
	// Push and Pop use pointer receivers because they modify the slice's length,
	// not just its contents.
	*h = append(*h, x.(int))
}

// 实现 Pop
func (h *PQ) Pop() interface{} {
	n := len(*h)
	x := (*h)old[n-1]
	*h = (*h)[:n-1]
	return x
}

// This example inserts several ints into an PQ, checks the minimum,
// and removes them in order of priority.
func main() {
	h := &PQ{2, 1, 5, 10, 4, 7}
	heap.Init(h)
	heap.Push(h, 3)

	for h.Len() > 0 {
		fmt.Printf("%d ", heap.Pop(h))
	}
}

以上就是 Golang 快速实现 Heap/Priority Queue 的一个模板，需要背下上面的代码。 但是在实际使用中，又会遇到下面 2 个问题

如何实现最大堆 ？#

上面的实现默认是生成一个最小堆，每次 Pop 都是返回堆中最小元素，那么如何生成一个最大堆呢 ？ 很简单，稍微修改一下把 Less() 函数即可

Less(i, j int) bool { return h[i] < h[j] }  // 最小堆
Less(i, j int) bool { return h[i] > h[j] }  // 最大堆

如何 Peek 堆顶元素 ？#

container/heap 的接口只有 Push 和 Pop，而有的时候我们只需要获取堆中的最大、最小元素，但是并不需要把它移除，这个时候该怎么办呢？

我们需要自己实践 Peek() 函数：根据前面堆的定义，我们知道数组的 0 号元素就是堆顶，因此直接返回 0 号元素即可。

func (h *PQ) Peek() int {
	return (*h)[0]
}

可以用下面的代码证明 0 号元素就是堆的最大/最小元素

package main

import (
	"container/heap"
	"fmt"
)

type PQ []int

// 实现 sort 接口
func (h PQ) Len() int           { return len(h) }
func (h PQ) Less(i, j int) bool { return h[i] < h[j] }
func (h PQ) Swap(i, j int)      { h[i], h[j] = h[j], h[i] }

// 实现 Push
func (h *PQ) Push(x interface{}) {
	// Push and Pop use pointer receivers because they modify the slice's length,
	// not just its contents.
	*h = append(*h, x.(int))
}

func (h *PQ) Peek() int {
	return (*h)[0]
}

// 实现 Pop
func (h *PQ) Pop() interface{} {
	n := len(*h)
	x := (*h)[n-1]
	*h = (*h)[:n-1]
	return x
}

func main() {
	h := &PQ{2, 1, 5, 6, 4, 3, 7, 9, 8, 0}

	fmt.Printf("init: %v\n\n", *h)

	heap.Init(h)
	fmt.Println(*h)

	for h.Len() > 0 {
		heap.Pop(h)
		fmt.Println(*h)
	}
}

由此可见，0 号元素就是最大或者最小元素。 练习题可见 Meeting Rooms II

特别需要注意的是，模板代码中，我们实现了 Push 和 Pop，但是我们并不会直接调用它们，我们是通过 heap.Push(), heap.Pop() 来增加和删除元素，而我们自己增加的 Peek 函数，则需要用 h.Peek() 来调用，因为它不属于 container/heap 接口函数，这一点要注意。

总之，Golang 在一些基本数据结构方面操作真是麻烦。

基于以上代码，利用 heap 库在实现一套复杂一点的 Priority Queue 就不难了，一个例子如下

// An Item is something we manage in a priority queue.
type Item struct {
	value    string // The value of the item; arbitrary.
	priority int    // The priority of the item in the queue.
	// The index is needed by update and is maintained by the heap.Interface methods.
	index int // The index of the item in the heap.
}

// A PriorityQueue implements heap.Interface and holds Items.
type PriorityQueue []*Item
